Made the WCS coordinate transform paths (``pixel_to_world``, ``world_to_pixel``,
``all_pix2world``, ``wcs_pix2world``, ``wcs_world2pix``, ``mix``) safe to call
concurrently on a shared ``WCS`` instance from multiple threads.  Previously,
each call performed a NaN-to-UNDEFINED / UNDEFINED-to-NaN round-trip on the
wcsprm C struct around the wcslib transform; under multithreading (and
especially under free-threaded CPython), the round-trip's transient writes to
sentinel-valued fields such as ``obsgeo``, ``mjdobs`` and ``mjdavg`` could be
observed by another thread reading them via the WCS APE 14 high-level API,
leading to spurious ``"Input values for mjd class must be finite doubles"``
errors.  The round-trip has been moved out of the transform paths; the
required ``wcsset`` call (which is the only writer) now happens eagerly under
the GIL via ``Wcsprm_cset`` and is short-circuited on subsequent calls by
``wcsenq``.
