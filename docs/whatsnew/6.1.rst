.. _whatsnew-6.1-ascii-default-int-columns-as-int64:

``io.ascii`` uses 64-integers by default for integer columns
============================================================

:mod:`~astropy.io.ascii` now uses a 64-bit integer field by
default when reading a column of integer numeric data. This changes the default behavior
on Windows and potentially 32-bit architectures. Previously on those platforms, table
columns with any long integers which overflowed the 32-bit integer would be returned
as string columns. The new default behavior is consistent with ``numpy`` v2 and ``pandas``.

.. _whatsnew-5.0-quantity-format-spec:

``Quantity.to_string`` supports ``format_spec`` for formatting
==============================================================

The :meth:`~astropy.units.Quantity.to_string` method now supports a ``format_spec`` parameter,
enabling direct usage of Python's format specification mini-language for precise quantity formatting.
This allows for explicit and consistent control over numerical representation, including fixed-point
and scientific notation, directly in the string output.

Previously, the method primarily relied on the ``precision`` parameter for format control, which dictated
the number of significant digits and was dependent on internal thresholds.``format_spec`` enables explicit
specification of formats such as fixed-point notation, scientific notation, or any custom format compatible
with the mini-language.

Example:

.. code-block:: python

    from astropy import units as u
    q = u.Quantity(12345, u.m)
    print(q.to_string(format_spec='.2e'))  # Outputs: '1.23e+04 m'
